## ⚡ Overview

**MiroFish** is an integrated intelligent system for risk management and project/research monitoring—designed to be used **before** actual implementation to minimize the risk of failure and wasted time.

The system generates numerous scenarios and personas that simulate your project within a comprehensive, realistic environment using live, continuous data to optimize outcomes. It monitors how project personas interact and coexist within this environment **before** you spend money or effort, only to discover a lack of market demand or that your research was misaligned.

Throughout this data-driven virtual journey, you will receive reports and guidance to refine your work. The system's roadmap is updated in real-time with data extracted from live sources—such as **X**, **Reddit**, **YouTube**, and various research and project platforms—to demonstrate your project's resilience or struggle against challenges as conditions evolve.

### Why MiroFish?

- **De-risk Decisions**: Validate ideas before committing resources
- **Market Validation**: Discover demand gaps before launch
- **Research Alignment**: Ensure your research direction matches real-world needs
- **Live Intelligence**: Continuously updated with real market signals

## 🔄 How It Works

1. **Project Definition**: Input your project concept, research topic, or business idea
2. **Environment Construction**: The system builds a realistic simulation using live data from X, Reddit, YouTube, and research platforms
3. **Persona Generation**: Thousands of AI personas with distinct characteristics interact with your project concept
4. **Simulation Execution**: Observe how your project performs under various market conditions and scenarios
5. **Intelligence Reports**: Receive detailed risk assessments, market feedback, and optimization recommendations
6. **Continuous Monitoring**: The roadmap evolves in real-time as live data updates market conditions

## 🚀 Quick Start

### Source Deployment (Recommended)

#### Prerequisites

| Tool | Version | Notes | Verify |
|------|---------|-------|--------|
| **Node.js** | 18+ | Frontend runtime | `node -v` |
| **Python** | ≥3.11, ≤3.12 | Backend runtime | `python --version` |
| **uv** | Latest | Python package manager | `uv --version` |

#### 1. Configure Environment

```bash
# Copy example configuration
cp .env.example .env

# Edit .env with your API keys
```

**Required Environment Variables:**

```env
# LLM API Configuration (OpenAI SDK compatible)
# Recommended: Aliyun Bailian qwen-plus: https://bailian.console.aliyun.com/
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Zep Cloud Configuration
# Free tier supports basic usage: https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

#### 2. Install Dependencies

```bash
# One-command installation (root + frontend + backend)
npm run setup:all
```

Or step by step:

```bash
# Install Node dependencies
npm run setup

# Install Python dependencies
npm run setup:backend
```

#### 3. Start Services

```bash
# Start both frontend and backend
npm run dev
```

**Service URLs:**
- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:5001`

**Individual startup:**

```bash
npm run backend   # Backend only
npm run frontend  # Frontend only
```

### Docker Deployment

```bash
# 1. Configure environment
cp .env.example .env

# 2. Pull and start
docker compose up -d
```

Ports: `3000` (frontend) / `5001` (backend)
